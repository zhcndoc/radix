# 组成颜色调色板

<Description>
	使用 Radix Colors 组成颜色调色板的指南。
</Description>

## 选择品牌色阶

Radix 提供了多个色阶供您选择作为品牌色或强调色。

为白色前景文本设计的色阶：

<ColorGrayPairsComplementary />

### 自定义品牌颜色

Radix Colors 并不打算进行自定义。它们旨在具有可访问性、平衡性和和谐性。任何自定义都可能破坏这些特性。

如果您需要自定义品牌颜色，我们建议在 Radix 色阶旁边添加自定义色阶。

例如，您可以使用 Radix Colors 作为灰色调和语义色阶，然后添加自定义品牌/强调颜色。

## 选择灰色调

Radix Colors 提供了一种纯灰色和几种带色灰色调。

<ColorGrays />

- `gray` 是纯灰色
- `mauve` 基于紫色调
- `slate` 基于蓝色调
- `sage` 基于绿色调
- `olive` 基于青柠色调
- `sand` 基于黄色调

### 中性搭配

如果您想要中性的感觉，或者想保持简单，`gray` 可以与任何色调或调色板很好地搭配。

### 自然搭配

或者，选择与您强调色最接近的色调饱和的灰色调。两者之间的差异微妙，但这将创造更丰富和和谐的氛围。

<ColorGrayPairs />

<Note>
	注意：如果您的项目使用了很多多彩的 UI 组件，例如徽章，请在
	应用程序背景中谨慎使用饱和的灰色，特别是在深色模式下。多彩的 UI 组件可能与您的饱和灰色背景色相冲突。
</Note>

## 选择语义色阶

对于大多数项目，您需要颜色来传达语义含义。以下是在西方文化中常见的有效搭配。

- **错误**：`red`，`ruby`，`tomato`，`crimson`
- **成功**：`green`，`teal`，`jade`，`grass`，`mint`
- **警告**：`yellow`，`amber`，`orange`
- **信息**：`blue`，`indigo`，`sky`，`cyan`

在许多情况下，您可能最终需要大多数色阶，无论出于何种原因。您的应用可能支持多人模式，您为每个用户分配一个颜色。您的应用可能有标记功能，用户为任务分配颜色。您的应用可能使用徽章来传达“待处理”或“已拒绝”的状态。

Radix Colors 是经过良好平衡的，并旨在和谐工作。因此，出于产品沟通，大多数颜色搭配都能正常使用。

## 选择文本色阶

步骤 11 和 12 分别针对低对比度文本和高对比度文本。根据您想要的氛围，您可以使用强调色阶或灰色调。

使用强调色阶将产生更丰富的氛围。

{

<Box my="5">
	<Text
		as="p"
		size="7"
		align="center"
		weight="bold"
		style={{ color: "var(--blue-12)" }}
	>
		这段文本是蓝色 12
	</Text>
</Box>

}

使用灰色调将产生更实用的氛围。

{

<Box my="5">
	<Text
		as="p"
		size="7"
		align="center"
		weight="bold"
		style={{ color: "var(--slate-12)" }}
	>
		这段文本是青石色 12
	</Text>
</Box>

}

两者之间的差异可能看似微妙，但在整页文本中可能产生巨大差异。

您可能想在营销网站中尝试使用强调色阶作为文本，在应用程序中尝试使用灰色调作为文本。
