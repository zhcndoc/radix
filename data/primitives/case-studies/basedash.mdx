# Basedash

<HeroQuote>
	借助 Radix Primitives，我们现在可以在几分钟内实现新组件，并在一天内发布它们。这使我们能够将更多工程资源投入到核心产品中。
</HeroQuote>

**您正在使用 Radix Primitives 的项目类型是什么？**

我们正在使用 Radix Primitives 为 Basedash 的主应用程序构建一个自定义设计系统。该项目已经上线，拥有定期使用该应用程序的用户。我们有 2 位工程师定期在设计系统上工作，并扩展应用程序的核心功能。

<br />

**Radix Primitives 为您和您的公司提供了哪些帮助？**

在采用 Radix Primitives 之前，我们对构建复杂组件（例如警报和上下文菜单）持谨慎态度，因为我们担心它们的开发会耗费数天时间。如今，借助 Radix Primitives，我们可以在几分钟内实现新组件，并在一天内发布它们。我们很少需要考虑可访问性，因为 Radix 能够很好地处理这一点。这让我们能够将更多工程资源投入到核心产品中。

<br />

**您对 Radix 开发者体验的看法是什么？**

Radix 组件的 API 非常流畅。开箱即用简单，但如果需要也可以完全自定义。我们选择 Radix 的主要原因之一是这些组件没有样式。这对于那些构建自己组件库的公司来说是个巨大的优势——再也不用纠结于默认库的样式，我们可以按自己的想法完全应用样式。

<br />

**您对 Radix 文档的看法是什么？**

这些文档是我们使用过的最好的文档之一。它们包含每个属性的完整类型定义，对于在 TypeScript 代码库中工作时非常有用。文档中包含大量示例（包括一个实时交互组件），以及关于可访问性的文档，这一点很好。
