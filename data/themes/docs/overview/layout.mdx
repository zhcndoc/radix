# 布局

<Description>正确处理布局问题。</Description>

## 布局组件

布局组件用于将布局职责与内容和交互分开。这是_关注点分离_，使您的应用程序易于维护和理解，理解这些原则对于有效构建您的界面至关重要。

### Box

<ThemesBoxExample />

[Box](/themes/docs/components/box) 是最基本的布局组件。Box 用于：

- 为子元素提供间距。
- 对内容施加尺寸限制。
- 控制 Flex 和 Grid 容器内的布局行为。
- 根据屏幕大小使用其响应式 `display` 属性隐藏内容。

### Flex

<ThemesFlexExample />

[Flex](/themes/docs/components/flex) 组件可以做 Box 所能做的一切，但附带一组额外的属性来沿着某个轴组织项目。它提供方便访问 CSS [flexbox 属性](https://css-tricks.com/snippets/css/a-guide-to-flexbox/) 的功能。

### Grid

<ThemesGridExample />

[Grid](/themes/docs/components/grid) 用于以列和行的方式组织内容。与 Box 和 Flex 类似，它旨在提供对底层 CSS [grid 属性](https://css-tricks.com/snippets/css/complete-guide-grid/) 的方便访问，而不需任何额外的魔法。

### Section

<ThemesSectionExample />

[Section](/themes/docs/components/section) 在页面内容的较大部分之间提供一致的垂直间距，创造出层次感和分隔感。它仅提供几个预定义的大小，以保持简单和一致。

### Container

<ThemesContainerExample />

[Container](/themes/docs/components/container) 唯一的职责是给其包装的内容提供一致的 `max-width`。像 Section 一样，它仅提供几个与常见断点和典型内容宽度相适应的预定义大小，以确保舒适阅读。

## 常见布局属性

每个布局组件都有自己的一组专门属性，以及一组共享的常见布局属性。所有布局属性都支持 [响应式对象值](/themes/docs/theme/breakpoints)。

### 内边距

内边距属性可以访问 [空间比例步骤](/themes/docs/theme/spacing) 或接受任何有效的 [CSS 内边距值](https://developer.mozilla.org/en-US/docs/Web/CSS/padding)。

```jsx
<Box p="4" />
<Box p="100px">
<Box p={{ sm: '6', lg: '9' }}>
```

<ThemesPropsTable defs="paddingPropDefs" />

### 宽度

宽度属性接受任何有效的 [CSS 宽度值](https://developer.mozilla.org/en-US/docs/Web/CSS/width)。

```jsx
<Box width="100px" />
<Box width={{ md: '100vw', xl: '1400px' }} />
```

<ThemesPropsTable defs="widthPropDefs" />

### 高度

高度属性接受任何有效的 [CSS 高度值](https://developer.mozilla.org/en-US/docs/Web/CSS/height)。

```jsx
<Box height="100px" />
<Box height={{ md: '100vh', xl: '600px' }} />
```

<ThemesPropsTable defs="heightPropDefs" />

### 定位

定位属性可以改变元素相对于文档正常流的放置方式。与往常一样，每个属性都接受相应的 CSS 值，并且可以使用 [空间比例步骤](/themes/docs/theme/spacing) 作为偏移值。

```jsx
<Box position="relative" />
<Box position={{ initial: "relative", lg: "sticky" }} />

<Box inset="4" />
<Box inset={{ initial: "0", xl: "auto" }} />

<Box left="4" />
<Box left={{ initial: "0", xl: "auto" }} />
```

<ThemesPropsTable defs="positionPropDefs" />

### Flex 子元素

每个布局组件都有用于控制其作为 Flex 容器子元素时样式的属性。

```jsx
<Box flexBasis="100%" />
<Box flexShrink="0">
<Box flexGrow={{ initial: "0", lg: "1" }} />
```

<ThemesPropsTable defs="flexChildrenPropDefs" />

### Grid 子元素

每个布局组件都有用于控制其作为 Grid 容器子元素时样式的属性。

```jsx
<Box gridArea="header" />

<Box gridColumn="1 / 3" />
<Box gridColumnStart="2">
<Box gridColumnEnd={{ initial: "-1", md: "3", lg: "auto" }} />

<Box gridRow="1 / 3" />
<Box gridRowStart="2">
<Box gridRowEnd={{ initial: "-1", md: "3", lg: "auto" }} />
```

## 外边距属性

外边距属性在大多数组件上可用，以提供元素周围的间距。它们并不专属于布局组件。

外边距属性可以访问 [空间比例步骤](/themes/docs/theme/spacing) 或接受任何有效的 [CSS 外边距值](https://developer.mozilla.org/en-US/docs/Web/CSS/margin)。

```jsx
<Button m="4" />
<Button m="100px">
<Button m={{ sm: '6', lg: '9' }}>
```

<ThemesPropsTable defs="marginPropDefs" />

在不渲染 HTML 节点或依赖其 Root 部分进行布局的组件上，外边距属性可能不可用。

## 独立使用

如果需要，可以仅使用 Radix Themes 的布局组件。只需确保 JavaScript 树摇动在您这边正常工作，并导入支持布局样式的 CSS：

```jsx
import "@radix-ui/themes/layout.css";
```

您仍然需要用 `Theme` 包裹您的应用，以提供空间比例和缩放因子的设置。
